# Contributing

Contributions must preserve the project trust model and repository dependency rules.

Before opening a pull request:

1. Identify the owning product and capability.
2. Add or update the normative specification.
3. Include positive, negative, adversarial, and replay tests.
4. State whether the change modifies semantics, evidence, assurance, or compatibility.
5. Run the complete local `just check` suite.

Cross-cutting protocol changes require an RFC. Stable checker changes require independent review from a domain maintainer and a trust-model maintainer.
