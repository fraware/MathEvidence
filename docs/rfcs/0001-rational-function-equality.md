# RFC 0001 — Rational-Function Equality with Explicit Definedness Conditions

## Status

**Accepted** (Phase 0). Reference capability for Milestone 1 implementation.

## Objective

Given a Lean equality between supported rational expressions over `ℚ`, produce a theorem establishing equality under every required denominator condition, using candidate normalization generated by Mathematica or an open backend and a solver-independent Lean checker.

## Supported syntax

- variables of type `ℚ`;
- integer and rational literals;
- `+`, `-`, `*`, unary negation;
- natural powers;
- division;
- equality.

## Rejected syntax

- transcendental functions;
- conditionals;
- arbitrary user functions;
- casts whose semantics are not explicitly supported;
- inverses over structures lacking the required field semantics;
- approximate numerals.

## Request

Required fields:

- capability: `algebra.rational_equality`;
- capability version;
- canonical variable declarations;
- left expression;
- right expression;
- known assumptions;
- requested claim: `soundResult`;
- resource policy;
- request digest.

## Candidate and evidence

The backend returns:

- normalized numerator of `lhs - rhs` after common-denominator construction;
- denominator factors whose nonzero status is required;
- optional factorization data;
- backend provenance;
- request digest.

The backend does not return a trusted Boolean.

## Checker

The checker:

1. Recomputes the rational-expression semantics.
2. Verifies request binding.
3. Verifies that the returned denominator set is sufficient for every division used by the derivation.
4. Verifies the polynomial numerator identity through Lean normalization.
5. Produces equality under the resulting nonzero assumptions.
6. Exposes assumptions not available in the local context as goals.

## Claim limitation

The result is equality on the explicitly defined domain. It does not establish equality of totalized field expressions at points where original denominators vanish unless a separate theorem proves that stronger claim.

## Backend conformance

Mathematica and the selected open backend must pass:

- valid identities;
- false identities;
- missing condition cases;
- redundant condition cases;
- request-hash mismatch;
- variable permutation;
- large coefficient limits;
- unsupported syntax rejection.

## Definition of done

- one shared Lean checker;
- two conforming backends;
- offline replay;
- stable tactic;
- adversarial suite;
- axiom audit;
- and real-world benchmark adoption.
